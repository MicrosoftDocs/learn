Azure Pipelines is a powerful service that helps you automatically build, test, and deploy your applications. It works with any Git provider and can deploy to most major cloud services, including Azure.

Think of Azure Pipelines as your automated assistant that takes care of repetitive tasks like:
- Building your code when you make changes.
- Running tests to catch problems early.
- Deploying your applications to different environments.

This module will help you understand what Azure Pipelines is, how its components work together, and how to make decisions about automating your development workflow.

## What you'll learn

By the end of this module, you'll be able to:

- **Understand Azure Pipelines** - Know what it is and how it helps your development team.
- **Explain key components** - Understand how different parts work together to automate your workflow.
- **Choose automation strategies** - Decide what tasks to automate and who should be responsible.
- **Use essential terminology** - Speak confidently about Azure Pipelines concepts and features.

## Before you start

To get the most from this module, it helps if you have:

- **Basic DevOps understanding** - Know what DevOps is and why teams use it.
- **Some Git experience** - Familiar with version control concepts (but not required).
- **Development background** - Experience working on software projects in any capacity.

Don't worry if you're new to pipelines - we'll start with the basics and build up your understanding step by step.

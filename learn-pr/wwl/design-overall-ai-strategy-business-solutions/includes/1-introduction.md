As organizations accelerate their adoption of artificial intelligence, solution architects play a pivotal role in defining strategies that unlock business value while maintaining operational excellence and governance. This module provides comprehensive guidance on implementing the AI adoption process as outlined in the Cloud Adoption Framework (CAF) for Azure, specifically tailored for architecting AI agents within enterprise environments.

The CAF framework integrates with the [Well-Architected Framework](/azure/well-architected/) and [Azure Architecture Center](/azure/architecture/).

## Why consolidate the CAF and agent lifecycle frameworks?

CAF offers an end-to-end backbone for AI solution adoption on Azure, establishing foundational and operational guardrails. Layering in the Agent Adoption guidance introduces a dedicated operating model for planning, governing, building, and operating AI agents. Unifying these frameworks mitigates risks such as agent sprawl, ensures robust governance, and accelerates value realization across the organization.

## Learning objectives

By the conclusion of this unit, you'll be able to:

- Map CAF's AI adoption phases—Strategy, Plan, Ready, Govern, Secure, Manage—to the AI agent lifecycle (Plan agents, Govern & secure agents, Build agents, Operate agents)
- Design an enterprise-ready operating model that aligns landing zones, data, security, and agent governance
- Select and apply the appropriate Microsoft platforms (Foundry, Copilot Studio, Azure AI services, Microsoft 365 Copilot) using decision criteria for agents versus applications
- Use actionable checklists to advance from proofs of concept to production operations for both AI workloads and agents

### Phase-by-phase guidance

You'll be introduced to a unified process map that links CAF's strategic and operational phases to the agent lifecycle, ensuring that each step—from business outcome definition and technology selection to governance enforcement and operational management—is methodically addressed.

## Implementation

- **Strategy & Planning:** Define business outcomes, prioritize use cases, and select the appropriate Microsoft AI technologies. Decide on agent applicability and platform choice (SaaS vs. custom), establishing measurable success metrics and Responsible AI strategies

- **Ready & Foundations:** Establish landing zones, resource organization, and agent governance boundaries. Prepare the data architecture to ensure agents operate on authoritative, governed sources

## Ongoing processes

- **Govern & Secure:** Enforce governance policies, monitor organizational risks, and implement agent-specific security controls

- **Build & Operate:** Standardize agent development processes, use reference architectures, and operationalize agent fleets with monitoring, cost controls, and lifecycle management

### Architectural and operational considerations

You'll learn how to design operating models that align with enterprise landing zones, data estates, and security frameworks.

**Azure enterprise landing zones** are pre-configured, secure, and scalable multi-account cloud environments designed to provide a foundational setup for deploying workloads. For more information, see [/azure/cloud-adoption-framework/ready/landing-zone/](/azure/cloud-adoption-framework/ready/landing-zone/)

**Azure data estates** are the comprehensive, integrated ecosystem of an organization's data assets, spanning on-premises, cloud, and hybrid environments, managed, secured, and analyzed using Microsoft Azure tools.

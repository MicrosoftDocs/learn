Securing data in Microsoft Fabric starts with choosing the right level of access for each user. In this module, you learned how Fabric's layered security model works — from broad workspace roles down to granular table and folder controls — and when to use each layer.

Use **workspace roles** when users need access to all items in a workspace. Use **item permissions** when they need access to a specific item, like a single lakehouse. And when they need access to specific tables or folders within that item, create **OneLake security roles** to define exactly what data they can see.

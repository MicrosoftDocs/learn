The Fabric data agent you explored in this module transforms how clinical staff at Lamna Healthcare interact with operational data. Rather than waiting for IT to run a query, staff can ask questions like "How many rooms in the Surgical Services department are currently occupied?" and get governed, accurate answers in seconds.

In this module, you learned how using an ontology as a data source enables a data agent to understand business concepts rather than raw database schemas. The ontology defines entity types — Departments, Hospitals, Patients, Rooms, and VitalSignEquipment — and the relationships between them, giving the agent the semantic context it needs to translate natural language into GQL queries. You learned how to connect an ontology as a data source, write agent instructions that guide interpretation, and use the steps dropdown to validate how the agent reasons through a question. You also learned how publishing creates a stable version for colleagues while the draft remains available for ongoing refinement, and why recipients need read access to both the ontology and its underlying data sources.

Fabric data agents remove the technical friction between questions and answers — no SQL required, no waiting for IT, and no risk of ungoverned data access.

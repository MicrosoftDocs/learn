## Update necessary infrastructure and tooling

Depending on the results of your assessment in the Plan phase, you might need to implement new or update your current infrastructure:

- Microsoft Intune
- Microsoft Entra ID
- Windows Autopatch
- Windows Autopilot
- Windows Update for Business Reports
- Endpoint Analytics
- Windows Hello for Business
- Microsoft Configuration Manager (Implement any site server updates required.)
- Delivery Optimization (have you [configured it appropriately](/windows/deployment/do/delivery-optimization-configure) for your environment?)

> [!TIP]
> You might also decide to update networking hardware and tools, servers, VPN software, security tools and agents, and update management tools like data loss prevention agents.

Work with the stakeholders identified in your RACI matrix to review changes to the infrastructure and update any existing tools.

If you’re adopting new infrastructure or tools, use the prerequisites guidance in the Plan module to acquire and activate new services.

> [!NOTE]
> For Microsoft Entra ID and Windows Hello for Business, you’ll want Security or Identity personnel to be responsible.

> [!NOTE]
> ***Recommended deliverable***
>
> Updates to infrastructure completed. Update your procedural documentation with the steps taken to prepare your infrastructure layer. Update your list of chosen and available infrastructure and tools. Check: Do you need to get stakeholder approvals?

## Validate infrastructure and tooling

Work with the stakeholders identified in your RACI matrix to test and implement changes to the infrastructure and any new infrastructure you introduced.

> [!TIP]
> Use some test devices to validate that data is flowing between the various systems as expected. Are you receiving data for reports? Can devices reach all the endpoints they need? Can you deploy applications?

> [!NOTE]
> ***Recommended deliverable***
>
> Update your procedural documentation with the steps taken to test and implement infrastructure changes. Check: Do you need to get stakeholder approvals?

## Compare pass or fail results against readiness criteria

You defined your infrastructure readiness criteria in the Plan module. Compare your infrastructure validations against this list and identify if you need to make any changes.

> [!NOTE]
> ***Recommended deliverable***
>
> Document any infrastructure and tooling changes or remediations that are required before you can move to the deployment phase. Check: Do you need to get stakeholder approvals?

## Remediate infrastructure and tooling

If you identified any required infrastructure or tooling changes in the previous validation step, now is the time to implement those changes and remediations. Then revisit the validation step to confirm that your infrastructure meets the readiness criteria.

> [!NOTE]
> ***Recommended deliverable***
>
> Document any infrastructure and tooling changes made and the final validation results. Check: Do you need to get stakeholder approvals?

## Sign off and update procedural documentation

Document the success, failure, and remediation of all steps you take to close the infrastructure readiness gaps.

As you make decisions throughout this stage, document them in an easily shareable format for tracking, reporting, and continuity purposes. Get approvals of these deliverables from all the people identified in your RACI matrix connected to user readiness. Seek help and address any gaps before moving to the deployment phase.

## Define user readiness criteria

In the next phase, you’ll prepare users for new Windows 11 capabilities to improve productivity. For now, let’s define what would make users ready for Windows 11.

User readiness criteria should include:

- User groups identified by business unit needs
- A pool of test users
- Business unit champions or representatives to support their area with the update
- A communication schedule to prepare different user groups for changes
- Support resources for different stages of the update
- Email templates

> [!TIP]
>
> Try to include helpdesk users in this planning phase.

## Evaluate user identity

Just like device identity, user identity is fundamental for managing access to resources and ensuring security. User identity protects sensitive information from unauthorized access and helps to meet regulatory requirements.

Working with the identity owner, review user identity components such as [authentication methods](/entra/identity/authentication/concept-authentication-methods) or [role-based access control (RBAC)](/entra/identity/role-based-access-control/) to ensure they meet your requirements. Consider [identity and access management (IAM)](https://www.microsoft.com/security/business/security-101/what-is-identity-access-management-iam).

> [!NOTE]
> ***Recommended deliverable:***
> 
> Curated list of identity components in use by your organization today. List of any changes needed to user identity.

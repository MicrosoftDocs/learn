Windows 365 delivers Windows desktops by hosting them on cloud infrastructure and streaming the experience to users' local devices via web browsers or other supported applications. This process is similar to how video streaming services deliver content, with the addition of real-time user interaction.

In this case, the Windows operating system runs in the cloud, while user input—such as keyboard and mouse actions—is sent from the device to the Cloud PC. Visual and audio desktop output is streamed back to the user, the user reacts, and the cycle of local input and cloud-based processing continues.

Each Cloud PC is persistent, meaning it retains the user's settings, applications, and data between sessions. When a user signs out, their desktop state is preserved, and when they sign back in, they return to the same environment. This persistence means the experience closely resembles using a traditional personal computer, even though the desktop is hosted remotely.

Windows 365 integrates with identity and access services to ensure only authorized users can access their Cloud PttCs. Users authenticate with their organizational credentials, and access policies help control when and how Cloud PCs can be used—disallowing usage over risky connections or from potentially compromised accounts, for instance.

From a management perspective, Windows 365 unifies desktop hosting, identity, and management capabilities in a single service. IT teams can define how Cloud PCs are provisioned and maintained without directly managing the physical devices users connect from. This separation between the desktop environment and the endpoint device is a core part of how Windows 365 works.

Importantly, Windows 365 is designed to deliver a familiar Windows experience rather than teach users new ways of working. Applications behave as they would on a local Windows desktop, and users interact with tools they already know how to use. This design helps organizations adopt cloud-based desktops while minimizing changes to workflows.

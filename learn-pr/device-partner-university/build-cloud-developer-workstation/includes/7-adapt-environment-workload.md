In the previous exercise, you confirmed that your Windows 365 Cloud PC is ready to use from the moment you sign in. You were able to access your tools, open a project, and begin working with little setup.

But different projects and workflows require different capabilities—and your environment needs to adapt with you. Windows 365 can scale up the "ready-to-code" experience by allowing organizations to tailor Cloud PCs to match the demands of specific workloads.

## Flexible compute for different scenarios

Not all development tasks require the same level of compute. The table below shows how different workflows map to different resource needs.

| **Workflow type** | **Typical scenarios** | **Compute needs** |
| :---: | :---: | :---: |
| **Lightweight workflows** | Editing code, running scripts, testing simple web applications | Lower CPU and memory requirements |
| **Intermediate workflows** | Building applications, running integration tests, working with moderate datasets | Balanced CPU and memory resources |
| **Compute-intensive workflows** | Large builds, data processing, simulations, or AI and machine learning scenarios | Higher CPU, increased memory, or GPU support (in supported configurations) |

This flexibility helps organizations match Cloud PC configurations to project requirements, improving performance while avoiding overprovisioning. It can give developers the resources they need without rebuilding or reconfiguring their environment.

## Specialized environments for advanced workloads

For demanding scenarios such as machine learning training, Cloud PCs can be configured with enhanced capabilities. These include:

- Higher CPU configurations for compute-intensive tasks
- GPU-enabled environments for graphics or AI processing in supported configurations
- Configurations with optimized performance for low-latency, interactive workflows, such as step-by-step execution for remote debugging

These options allow teams to align environments with the specific requirements of their projects while maintaining a stable developer experience.

## Consistency with flexibility

Even as environments scale and adapt, they remain:

- **Standardized**, so teams work from a consistent baseline
- **Managed**, so IT can maintain security, compliance, and governance
- **Ready-to-use**, so developers reduce configuration overhead

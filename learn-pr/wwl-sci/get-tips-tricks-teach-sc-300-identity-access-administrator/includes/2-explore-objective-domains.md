


All topics covered in the [SC-300: Microsoft Identity and Access Administrator](https://aka.ms/SC300exam) course are related to identity and access administration using Microsoft Azure Active Directory (Azure AD) and map directly to [Exam SC-300: Microsoft Identity and Access Administrator](https://github.com/MicrosoftLearning/SC-300-Identity-and-Access-Administrator/blob/master/Instructions/Labs/Lab_13_ImplementAndTestAConditionalAccessPolicy.md). Percentages indicate the relative weight of each area on the exam. The higher the percentage, the more questions you're likely to see in that area. For the most up-to-date breakdown, access the [SC-300 study guide](https://github.com/MicrosoftLearning/SC-300-Identity-and-Access-Administrator/blob/master/Instructions/Labs/Lab_17_DefenderForCloudAppsDiscoveryAndRestrictions.md) from the [Exam SC-300: Microsoft Identity and Access Administrator](https://github.com/MicrosoftLearning/SC-300-Identity-and-Access-Administrator/blob/master/Instructions/Labs/Lab_26_ConfigurePrivilegedIdentityManagementForAADRoles.md) page.

Become familiar with the objective domains, which are a comprehensive set of specific and measurable knowledge, skills, and abilities that are the basis for the development of both this course and the certification exam:

- Module 1: Explore identity in Azure AD
- Module 2: Implement an identity management solution
- Module 3: Implement an authentication and access management solution
- Module 4: Implement access management for apps
- Module 5: Plan and implement an identity governance strategy

Remember that each module points back to the core theme of identity and access administration using Microsoft Azure Active Directory (Azure AD). Details for each module are included in the corresponding module best practice sections.

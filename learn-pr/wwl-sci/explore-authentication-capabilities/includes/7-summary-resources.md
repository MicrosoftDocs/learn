In this module, you learned about the authentication methods available in Microsoft Entra ID, including passwords, phone-based methods, OATH tokens, and passwordless options like Windows Hello for Business, passkeys (FIDO2), Microsoft Authenticator, and certificate-based authentication. You learned why phishing-resistant authentication methods provide the most secure sign-in experience, especially against AI-driven cyberattacks.

You learned about multifactor authentication and how it improves security by requiring two or more verification methods. You explored self-service password reset (SSPR), which reduces IT costs by enabling users to reset their own passwords, and account recovery, which uses AI-powered identity verification to help users who have lost access to all authentication methods.

You also learned how Microsoft Entra password protection mitigates the risks of weak passwords through global and custom banned password lists, and how the password evaluation process uses normalization and fuzzy matching to block weak passwords and their variations.

Now that you've completed this module, you can:

- Describe the authentication methods of Microsoft Entra ID.
- Describe multifactor authentication (MFA) in Microsoft Entra ID.
- Describe self-service password reset (SSPR) in Microsoft Entra ID.
- Describe the password protection and management capabilities of Microsoft Entra ID.

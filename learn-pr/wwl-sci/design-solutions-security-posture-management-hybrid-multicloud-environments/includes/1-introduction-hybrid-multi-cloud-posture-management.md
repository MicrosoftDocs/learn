Organizations operate infrastructure across on-premises datacenters, multiple public clouds, and edge locations. This distributed architecture makes it difficult to maintain consistent visibility into security risks and enforce unified policies when workloads span Azure, AWS, Google Cloud, and private infrastructure.

Suppose you work as a security architect at a financial services organization. Your company runs production workloads in Azure, legacy applications on-premises, and recently acquired a subsidiary that uses AWS. You need to design a solution that provides unified security posture management across all these environments—answering questions like: What's our overall security posture? Where are our most critical vulnerabilities? Which misconfigurations could an attacker exploit to reach sensitive data?

This module guides you through designing that solution using Microsoft Defender for Cloud, Azure Arc, and Microsoft Security Exposure Management.

## Learning objectives

By the end of this module, you're able to:

- Evaluate security posture by using Microsoft Cloud Security Benchmark, Microsoft Defender for Cloud, and secure scores
- Design integrated security posture management and workload protection solutions in hybrid and multicloud environments
- Design cloud workload protection solutions that use Microsoft Defender for Cloud
- Design solutions for Microsoft Defender External Attack Surface Management (Defender EASM)
- Specify requirements for secure posture management using Microsoft Security Exposure Management attack paths

## Prerequisites

- Advanced experience and knowledge in identity and access, platform protection, security operations, securing data, and securing applications
- Experience with hybrid and cloud implementations

Azure Functions is an event-driven function as a service (FaaS) offering. Developers write focused functions that are invoked by triggers and can use bindings to connect to other services. Typically, these functions are stateless and short-lived. Backend as a service (BaaS) refers to managed backend capabilities, such as storage, databases, messaging, identity, and APIs, that your code can use without managing servers. In an IoT application, Azure Functions can process events from services such as IoT Hub, Event Hubs, Azure Event Grid, and Azure Blob Storage, and can call Foundry Tools or store results in Azure Cosmos DB.

Azure Functions provides a way to create IoT solutions that you can rapidly deploy. Using the serverless computing approach, you can focus on the core logic of your solutions. You avoid much of the work of managing the underlying infrastructure that runs your solution. In the serverless application model, the cloud service provider provisions, scales, and manages the infrastructure required to run the code according to the hosting plan and service configuration. The serverless model commonly includes **function as a service** and **backend as a service**.

Billing and scaling behavior depend on the Azure Functions hosting plan. Flex Consumption is the recommended serverless pay-as-you-go hosting plan for new function apps. In that plan, Azure Functions scales automatically and charges based on executions and resources used. Premium, Dedicated, and Container Apps hosting plans have different billing, scaling, networking, and timeout characteristics.

When you choose Flex Consumption, the recommended serverless pay-as-you-go plan for new function apps, you don't have to provision fixed compute capacity in advance. The Consumption plan is the legacy serverless hosting option. It's currently the only serverless hosting option for Windows function apps, but [Linux Consumption hosting retires on September 30, 2028](/azure/azure-functions/consumption-plan), so existing Linux Consumption apps should [migrate to Flex Consumption](/azure/azure-functions/migration/migrate-plan-consumption-to-flex). Other hosting plans have different billing, scaling, and capacity behavior. Using this approach, you can create scalable IoT applications where devices send data to an ingestion service, such as IoT Hub or Azure Blob Storage, and Azure Functions runs event-driven business logic in Azure.

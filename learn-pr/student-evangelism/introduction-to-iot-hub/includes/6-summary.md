Consider the sample smart city scenario we've considered. The city is trying to optimize traffic using data from sensors and cameras. The solution needs to extend beyond the basic telemetry service. You need a central messaging hub for bidirectional communication in the cloud. You'll need to authenticate, connect, and manage millions of devices securely. Additionally, you'll need to monitor device health over time to support availability and resiliency goals. Meeting those goals also requires device and application retry/reconnect logic and disaster recovery planning. In this situation, IoT Hub can help solve the developer's problem because IoT Hub provides the framework to create supporting functions such as secure device connectivity, messaging, and device management. 

In the absence of the IoT Hub, the developer would have to create all the functionality themselves. By using IoT Hub in a cloud-based IoT solution, you can focus on the business problem at hand (providing city-wide solutions). IoT Hub simplifies IoT solution development by providing secure device connectivity, messaging, monitoring, and device-management capabilities at scale.

Solving city-wide problems requires a different type of thinking. An engineering-led approach based on systems thinking can provide solutions to complex problems like traffic management, reducing greenhouse-gas emissions from traffic, and improving safety on public transport. Solutions based on systems thinking need the ability to manage a large number of device-to-cloud and cloud-to-device messages between devices and the cloud back end through IoT Hub. IoT Hub provides a managed message hub for those communication patterns.

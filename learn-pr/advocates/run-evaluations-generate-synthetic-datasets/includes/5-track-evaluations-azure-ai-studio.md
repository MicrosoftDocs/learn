The Azure AI Evaluation SDK can log a run to your Microsoft Foundry project when you pass the `azure_ai_project` argument to `evaluate()`. Logging the run lets you inspect aggregate metrics, drill into row-level results, and compare multiple runs side by side in the portal. The AI-assisted quality evaluators still use your `model_config` deployment, while the logged run and safety evaluators use project-backed services.

## Scenario

Contoso Gameworks is developing an AI-powered dialogue generator for video game characters, customizing interactions based on game scenarios. The generator should be evaluated for relevance (fitting dialogue to the game's plot and character traits), fluency (smooth, immersive conversations), and risk and safety (ensuring no violent, offensive, or unfair language is introduced).

## Instructions

In this exercise, you run an evaluation to assess a dataset of character dialogue generated by the generator. You also log the results to your Foundry project so that you can review them in the portal. Open the `evaluate-track-results.ipynb` file to get started.

> [!NOTE]
> The notebook uses the local Azure AI Evaluation SDK workflow. If this is your first time logging a run to the project, make sure the project storage prerequisites are in place before you run `evaluate()`. For details, see [Evaluate your generative AI application locally with the Azure AI Evaluation SDK](/azure/foundry-classic/how-to/develop/evaluate-sdk#local-evaluation-on-test-datasets-using-evaluate).

> [!TIP]
> When `azure_ai_project` is set, `evaluate()` returns a result whose `studio_url` property is a deep link to the run in Microsoft Foundry. Open it after the call completes to inspect aggregate metrics and per-row scores.

For more information about reviewing runs in the portal, see [View evaluation results in the Microsoft Foundry portal](/azure/foundry/how-to/evaluate-results).

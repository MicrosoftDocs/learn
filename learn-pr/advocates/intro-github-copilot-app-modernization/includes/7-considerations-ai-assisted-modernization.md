Whilst GitHub Copilot app modernization provides you with ways to reduce the effort modernizing code, like any AI tool it isn't infallible. When modernizing critical software, it's important to remain vigilant. Some best practices include:

- **Always review the agent's output.** The agent generates assessment reports, plans, and code changes. Review each stage before proceeding. The three-stage workflow of assessment, planning, and execution is designed to keep you in control.
- **Ensure correctness with tests.** The agent automatically runs builds and tests, but you should also run your full test suite and perform manual testing, especially for edge cases. If the test coverage in the existing code that you want to migrate is low, ask the agent to generate additional tests before starting the upgrade. This allows you to verify that upgraded parts of the code provide the same test output as the existing code.
- **Check for outdated or suboptimal suggestions.** The GitHub Copilot app modernization agent's knowledge comes from training data and predefined task libraries. It might occasionally suggest an approach that isn't the latest best practice. For example, it might not automatically use the newest encryption algorithm or the most current Azure SDK method. Verify critical suggestions against current documentation.
- **Use predefined tasks for common scenarios.** Predefined tasks encode proven migration patterns. When a predefined task exists for your scenario, such as migrating from MSMQ to Azure Service Bus, use the predefined task rather than asking the agent to determine the migration path from first principles.
- **Create custom tasks for organizational patterns.** If your organization has specific coding standards or migration patterns, create custom tasks from successful prior migrations. This ensures consistency across teams and projects for modernized code.
- **Maintain consistency with code reviews.** Apply the same code review standards to agent-generated code as you would to human-written code. If working in a team, include AI-assisted changes in your normal pull request workflow.
- **Respect confidentiality constraints.** GitHub Copilot app modernization doesn't retain code snippets beyond the immediate session and doesn't collect or store your custom tasks. However, always follow your organization's policies regarding AI tool usage.
- **Know when to intervene manually.** If the agent consistently produces incorrect results for a particular section, it may be faster to handle that part manually. You can always resume the agent workflow for the remaining tasks.

Good security practices require that we control access to applications that we develop, and developing an application with Azure OpenAI in Microsoft Foundry Models is no different. In this module you'll learn about configuring authentication to Azure OpenAI in Microsoft Foundry Models using .NET, the available roles for Azure OpenAI in Microsoft Foundry Models, and the benefits of using managed identity over API keys.

The RBAC roles covered in this module apply to Azure OpenAI in Microsoft Foundry Models resources (the classic Azure OpenAI resource type), not Microsoft Foundry projects.

Learning objectives:

By the end of this module, you're able to:

- Describe the different Role Based Access control (RBAC) roles for use in Azure OpenAI in Microsoft Foundry Models and understand how to configure role assignments
- Understand the differences between using API keys and managed identities within Azure OpenAI in Microsoft Foundry Models
- Understand how to configure keyless authentication to Azure OpenAI in Microsoft Foundry Models for local development for .NET applications
- Understand how to configure keyless authentication to Azure OpenAI in Microsoft Foundry Models for Azure-hosted environments for .NET applications
